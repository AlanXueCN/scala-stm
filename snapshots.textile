Changes between 0.3 and 0.4-SNAPSHOT:
* CommitBarrier added, which allows multiple atomic blocks (each on its own thread) to commit together.
* Small performance improvements.
* STMBench7 benchmark support added.
* Automatic selection of STMImpl in most cases.

Snapshot releases deployed to the @scala-tools.org@ repository are fully tested and functional, but may have changing APIs.

h2(#link). Links

* Manual download: ["scala-stm_2.9.1-0.4-SNAPSHOT.jar":http://scala-tools.org/repo-snapshots/org/scala-tools/scala-stm_2.9.1/0.4-SNAPSHOT/scala-stm_2.9.1-0.4-SNAPSHOT.jar]
* ScalaDoc: ["0.4-SNAPSHOT":api/0.4-SNAPSHOT/index.html#scala.concurrent.stm.package]

h2(#sbt). SBT

Tell @sbt@ about the @scala-tools.org@ snapshot repository and the ScalaSTM artifact by adding the following two lines to your project definition class (in @project/build@)

{% highlight scala %}
val scala_tools_snapshots = ("Scala-Tools Maven2 Repository - snapshots" at
    "http://scala-tools.org/repo-snapshots")
val scala_stm = "org.scala-tools" %% "scala-stm" % "0.4-SNAPSHOT"
{% endhighlight %}

@sbt update@ will download the latest @scala-stm@ snapshot JAR and make it available for building.

h2(#maven). Maven2

The ScalaSTM snapshot dependency for your @pom.xml@ is

{% highlight xml %}
<dependencies>
  <dependency>
    <groupId>org.scala-tools</groupId>
    <artifactId>scala-stm_2.9.1</artifactId>
    <version>0.4-SNAPSHOT</version>
  </dependency>
</dependencies>
{% endhighlight %}

Make sure you have the @org.scala-tools@ snapshot repository available

{% highlight xml %}
<repositories>
  <repository>
    <id>scala-tools.org.snapshots</id>
    <name>Scala-Tools Maven2 Repository - snapshots</name>
    <url>http://scala-tools.org/repo-snapshots</url>
    <snapshots/>
  </repository>
</repositories>
{% endhighlight %}

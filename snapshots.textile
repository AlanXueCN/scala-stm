Changes between 0.2 and 0.3-SNAPSHOT:

* Bug fixes (["#11":https://github.com/nbronson/scala-stm/issues/closed#issue/11], ["#12":https://github.com/nbronson/scala-stm/issues/closed#issue/12], ["#13":https://github.com/nbronson/scala-stm/issues/closed#issue/13])

Snapshot releases deployed to the @scala-tools.org@ repository are fully tested and functional, but may have changing APIs.

h2(#link). Links

* Manual download: ["scala-stm_2.8.1-0.3-SNAPSHOT.jar":http://scala-tools.org/repo-snapshots/org/scala-tools/scala-stm_2.8.1/0.3-SNAPSHOT/scala-stm_2.8.1-0.3-SNAPSHOT.jar]
* ScalaDoc: ["0.3-SNAPSHOT":api/0.3-SNAPSHOT/index.html]

h2(#sbt). SBT

Tell @sbt@ about the @scala-tools.org@ snapshot repository and the ScalaSTM artifact by adding the following two lines to your project definition class (in @project/build@)

{% highlight scala %}
val scala_tools_snapshots = ("Scala-Tools Maven2 Repository - snapshots" at
    "http://scala-tools.org/repo-snapshots")
val scala_stm = "org.scala-tools" %% "scala-stm" % "0.3-SNAPSHOT"
{% endhighlight %}

@sbt update@ will download the latest @scala-stm@ snapshot JAR and make it available for building.

h2(#maven). Maven2

The ScalaSTM snapshot dependency for your @pom.xml@ is

{% highlight xml %}
<dependencies>
  <dependency>
    <groupId>org.scala-tools</groupId>
    <artifactId>scala-stm_2.8.1</artifactId>
    <version>0.3-SNAPSHOT</version>
  </dependency>
</dependencies>
{% endhighlight %}

Make sure you have the @org.scala-tools@ snapshot repository available

{% highlight xml %}
<repositories>
  <repository>
    <id>scala-tools.org.snapshots</id>
    <name>Scala-Tools Maven2 Repository - snapshots</name>
    <url>http://scala-tools.org/repo-snapshots</url>
    <snapshots/>
  </repository>
</repositories>
{% endhighlight %}
